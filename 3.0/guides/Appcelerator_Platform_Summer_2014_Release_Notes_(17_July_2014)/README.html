Upgrade Guide

Upgrade your Environmen t

On each launch, Studio automatically checks if an update is available and presents an alert you can click to start the install process. To manually install the updates at any time, select Help > Check for Appcelerator Updates. When the update is complete, the following components will be updated to their released version:

  • Appcelerator Studio

  • Titanium SDK

  • Node.ACS CLI (command-line tool)

  • Appcelerator Performance module for Studio

Appcelerator Dashboard is updated automatically to version 2.0. If you experience any weird behavior, please logout and log back in.

Upgrade your app to use the latest SDK

This release of the Titanium SDK includes the following behavior and feature changes, which may require you to update your application.

For Android applications:

  1. Due to the addition of the appcompat library, there are a number of behavior changes to the applications:

    1. The action bar is now enabled by default. Previously, you had to set a theme to enable the action bar.

    2. The target SDK must be set to API level 14 (Android 4.0.x) or higher, or you must have API level 14 or higher installed.

    3. If the application is using a custom theme, the parent theme must be based off an AppCompat theme. The application cannot use the base Holo, Light or Dark themes.

  2. All windows created are heavyweight. There is no more support for lightweight windows. Due to this change, your application may behave differently or break.

  3. The Android-specific enabled property for view components is deprecated. Use the non-platform-specific touchEnabled property to disable touch events for a view component.

For iOS applications, to achieve parity across platforms, the Titanium.Database.ResultSet.fieldcount() method has been removed from the SDK. Use the read-only Titanium.Database.ResultSet.fieldcount property to retrieve the same information.

For all Titanium applications, the Titanium.include method is deprecated and will removed in Release 3.4.0. To include an external JavaScript file, create a CommonJS module.

Track test success rate

Measuring the success rates of your tests is critical to the quality of your application. The Test success rate metric:

  • measures the number of tests completed successfully against all test run
  • is available for the entire app portfolio or a single app
  • is available by supported platforms (iOS, Android) to give you an estimate of which platforms show the more problems

Other test metrics such as the number of tests run and the status of tests run are also available.

2-factor authentication using SMS

Security has been a key priority for us. You store a lot of business critical information in the Appcelerator Cloud and we want to ensure that no unauthorized person gets access to it.

We have enhanced our login process with a complete 2-factor authentication mechanism using both email and SMS. Every user is prompted for his phone number on account creation and has the option to receive his device authorization code on his mobile phone. For existing users, just go in your profile and enter a phone number to enable 2-factor authentication

Dashboard support for Virtual Private Cloud deployments

We have received a lot of feedback on how to make the Appcelerator Dashboard and Studio work better with Virtual Private Cloud deployments. Some of the features included in this release are:

  • Use the Dashboard environment filters to manage your cloud in the various Virtual Private Cloud instances. Where before you only saw Development and Production, if you are a VPC customer, you'll now see all the VPC environments configured for your organizations. Learn more

  • Studio is aware of your VPC environments. So now, you can create, develop and publish custom APIs directly into one of your VPC environments. You can then switch to the Dashboard to view their analytics.

  • When you create a new mobile app from Studio, all pre-built APIs are automatically created for you in your respective VPC environments. As with custom APIs, you can use the dashboard's environment filters to manage and view logs for these APIs in each VPC environment. Learn more

Note: Please contact Appcelerator Support if you do not see your VPC environments in the Dashboard filters

Pre-built Cloud APIs

Below are some featured highlights in this Appcelerator Cloud release. See the ACS 1.1.3 release notes for a full list of fixes and changes.

Create a Files object using a remote URL

The Files create method now includes a url field, so you can specify a remote file to upload to the Appcelerator Cloud servers.

Prettify JSON response output

Each ACS method request now includes the pretty_json field to enable or disable prettifying JSON response data. By default, JSON pretty print is disabled.

Previously, JSON response output was always prettified.

Query likes by user

You can now query liked object by user using the the Likes query method. Use this method to retrieve a list of objects the current user liked.

Titanium SDK 3.3.0

Below are some featured highlights in this Titanium SDK release. See the Titanium SDK 3.3.0 release notes for a full list of fixes and changes.

AppCompat Library Support

As of this Release, the Titanium SDK uses the appcompat library to provide support for the Action Bar on devices running Android 2.3.x and later. Due to the addition of the appcompat library, there are a number of behavior changes to the application:

  1. The action bar is now enabled by default. Previously, you had to set a theme to enable the action bar.

  2. The target SDK must be set to API level 14 (Android 4.0.x) or higher, or you must have API level 14 or higher installed.

  3. If the application is using a custom theme, the parent theme must be based off an AppCompat theme. The application cannot use the base Holo, Light or Dark themes.

For more information about using Android Themes and the Action Bar, see Android Themes and Android Action Bar, respectively.

Core Motion Module

The Core Motion module provides support for monitoring various hardware sensors on iOS devices, such as the accelerometer, gyroscope, and magnetometer. The Core Motion module allows you to access the measurements provided by these sensors.

For more information, see the Core Motion Module guide and CoreMotion API Reference.

New HTTP client module based on NSURLConnection

The HTTPClient module on iOS has been rebuilt on the iOS NSURLConnection class, which provides improved security and other benefits over the previous implementation. Existing projects will work automatically with the new module without any code changes.

SSL Certificate Pinning Module

SSL certificate pinning is a technique to prevent so-called "man-in-the-middle" attacks in which the attacker spoofs a server&rsquo;s SSL certificate. With certificate pinning, you embed the host's SSL certificate in your Titanium application. For more information, see the HTTPS Module documentation.

Clip a view or add a shadow to a view

This release introduces several new properties to control the clipping behavior of a view or add a drop shadow to a view. Use the clipMode property to control the clipping behavior of the view component.

To add a shadow to a view component, use the following properties to control the appearance of the shadow:

  • viewShadowColor: sets the color of the shadow

  • viewShadowOffset: sets the offset of the shadow

  • viewShadowRadius: set the radius of the rounded corners of the shadow

Windows Hybrid Support

This release supports Windows 8 Apps and Windows Phone 8 applications built using the Mobile Web APIs. Titanium turns your Mobile Web application in to Windows 8 JavaScript and Windows Phone HTML5 applications.

All development is done through the command-line tools. There is no support with Appcelerator or Titanium Studio.

For information on getting started, see:

  • Getting Started with the Windows Phone SDK

  • Getting Started with the Windows 8 SDK

To access native features of the Windows Phone platform, see Windows Phone 8 Module Development Guide.

Content Assistance improvements

This release includes the following content assistance improvements:

  • Add code assistance for methods that take constants as arguments. For example, suggests Ti.Media.CAMERA_FRONT orTi.Media.CAMERA_REAR for the Ti.Camera.swtichCamera method.

  • Add code assistance for proxy properties for XML elements in Alloy projects. For example, suggests the LeftNavButton or RightNavButton elements as children of the Window element.

  • Add code assistance for APIs in the Alloy namespace. For example, suggests Alloy.Globals or Alloy.CFG if only Alloy. is typed.

  • Add hovers (info box when hovering on an API element) for XML elements, XML attributes and TSS attributes for Alloy projects.

Genymotion emulator support

Studio can now launch your application on a Genymotion virtual device. See Installing Genymotion for more information.

New Windows installer

The Windows installer now installs any missing dependencies required by Studio, including the Oracle JDK, Node.js and Git.

Alloy 1.4

Below are some featured highlights in this Alloy release. See the full list of changes and fixes in Alloy 1.4.

Custom TSS queries

You can now apply custom logic to TSS queries. For example, in the following example the #textLabel style is applied only if Alloy.Globals.foo evaluates to true.

// index.tss
'#textLabel[if=Alloy.Globals.foo]': {
color: 'green',
}

Declaratively annotate maps

<Annotation> elements are supported within ti.map element for easy annotation and collection-view binding:

<Module id="map" module="ti.map">
<Annotation title="Appcelerator HQ" latitude="37.389569" longitude="-122.050212" />
</Module>

ActionBar properties on Menu elements

ActionBar properties be specified as attributes on a <Menu> element:

<Menu title="New Title" />

Widgets support themes

Alloy widgets now support theming for quick and easy styling changes.
