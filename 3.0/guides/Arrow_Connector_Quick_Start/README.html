Introduction

This guides covers the basic to create and build an Arrow Connector. An Arrow Connector is a component of an Arrow project that allows you to access data from an external source. An Arrow Connector is setup the same as an Arrow Builder project except you have logic for your connector in the lib folder, and do not have Arrow API, Block or Web components.

For this example, the connector will integrate with the Twitter REST API. You will need to create a new Twitter app and generate the OAuth access tokens. To create a new Twitter app, go to https://apps.twitter.com/, then generate the access token from the app's Keys and Access Tokens tab.

Create a Connector

To create a new connector, from your workspace directory, execute the appc generate command. When prompted, select Arrow Component for the type of component, Arrow Connector for the Arrow component to generate and Empty Connector Project for the connector to generate. Enter com.connector.twitter as the name and directory name for your project.

Add Dependencies

As with all NPM modules, add your dependencies to the dependencies object in the package.json file. This example connector will be utilizing the twitter NPM module to make calls to Twitter. Add the twitter key with a ~1.2.5 value to the dependencies object.

package.json
{ ...
"dependencies" : {
...
"twitter": "~1.2.5"
}
...
}

Configuration File

The boiler plate sample contains the conf/default.js configuration file. This file is used for testing the connector. You will also need to create a configuration file called example.config.js. When you install the connector, this file is copied the the Arrow project's conf directory. You need to add settings to configure your connector in these files, specifically add the settings to the connectors object. For the Twitter connector, it will need the app keys, OAuth tokens and an account name of the Twitter feed to access. Create the conf/example.config.js file, then add the content below to it. You will also need to add the following keys to the connectors object in the conf/default.js file and add your Twitter app information to it for testing purposes.

module.exports = {
connectors: {
'com.connector.twitter': {
account: 'TWITTER_ACCOUNT',
consumer_key: 'TWITTER_API_KEY',
consumer_secret: 'TWITTER_API_SECRET',
access_token_key: 'TWITTER_ACCESS_TOKEN',
access_token_secret: 'TWITTER_ACCESS_TOKEN_SECRET'
}
}
}

Initializer File

The app.js file contains code that initializes the connector when it is used as a server for testing. You can hook into the lifecycle events of the server. The boiler plate file contains logic that is used to create a user model and preload data. Remove the code between the server started event listener and server.start() method.

Model File

To access data from our connector, define a model file to generate the API endpoints and schema for the data. Replace the contents of model/test.js with the contents below. The boiler plate model file now uses the api/tweet endpoint instead of api/test, and the application can access the tweet's text response parameter as the status parameter.

model/test.js
var Arrow = require('arrow');
var Test = Arrow.Model.extend('tweet', {
fields: {
status: { type: String, name: 'text' }
},
connector: 'com.connector.twitter'
});
module.exports = Test;

Connector Logic

The lib/index.js file contains boiler plate code that implements a simple in-memory connector. The connector logic is simply a JavaScript file that exposes a create() method that returns an extended Arrow Connector object. The Connector object contains logic to implement CRUD methods for accessing the external data as well as configuration, setup and shutdown logic for the external data source.

There must be an index.js file in your lib folder, which is the first file loaded by the connector. You can have additional JavaScript file, which can be loaded by the lib/index.js file.

lib/index.js
...
// Load the twitter module
var Twitter = require('twitter'),
client = null;
 
exports.create = function(Arrow, server) {
var ORMError = Arrow.ORMError;
return Connector.extend({
...
// Tells the Appcelerator CLI where the default configuration file is
defaultConfg: require('fs').readFileSync(__dirname + '/../conf/example.config.js', 'utf8'),
// Modify the connect function to initialize the Twitter client
connect: function(callback) {
this.logger.debug('connecting');
// Initialize the client
// Use this.config to get values from the configuration file
client = new Twitter({
consumer_key: this.config.consumer_key,
consumer_secret: this.config.consumer_secret,
access_token_key: this.config.access_token_key,
access_token_secret: this.config.access_token_secret
});
this.logger.debug('connected');
callback();
},
...
// Overwrite the other boiler plate CRUD methods with the following...
create: function (Model, values, callback) {
var params = {'status': values.status};
client.post('statuses/update', params, function(error, tweet, response) {
if (!error) {
callback(null, createInstance(Model, tweet));
} else {
this.logger.error(error);
callback(new ORMError('ERROR: Unable to create tweet!'));
}
});
},
findAll: function (Model, callback) {
var params = {screen_name: this.config.account};
// this Twitter API only returns the last twenty tweets
client.get('statuses/user_timeline', params, function(error, tweets, response) {
if (!error) {
var results = [];
for (var i = 0; i < tweets.length; i++) {
results.push(createInstance(Model, tweets[i]));
}
callback(null, new Collection(Model, results));
} else {
this.logger.error(error);
callback(new ORMError('ERROR: Could not fetch tweets!'));
}
});
},
findOne: function (Model, id, callback) {
client.get('statuses/show/' + id, {}, function(error, tweet, response) {
if (!error) {
callback(null, createInstance(Model, tweet));
} else {
this.logger.error(error);
callback(new ORMError('ERROR: Could not fetch tweet!'));
}
});
},
query: function (Model, options, callback) {
callback(new ORMError('NOOP'));
},
save: function (Model, instance, callback) {
// Twitter does not have a method to update a tweet
// You can delete the tweet, then create a new one
callback(new ORMError('NOOP'));
},
'delete': function (Model, instance, callback) {
client.post('statuses/destroy/' + instance.id + '.json', {}, function(error, tweet, response) {
if (!error) {
callback(null, createInstance(Model, tweet));
} else {
this.logger.error(error);
callback(new ORMError('ERROR: Could not delete tweet!'));
}
});
},
deleteAll: function (Model, callback) {
callback(new ORMError('NOOP'));
}
});
};
/*
* Helper function to convert the raw data to a model
* and set the primary key to use the tweet's id_str
*/
function createInstance (Model, data) {
var model = Model.instance(data, true);
model.setPrimaryKey(data.id_str);
return model;
}

Test the Connector

Like an Arrow Builder project, you can locally run the Arrow Connector project and make APIs calls to it. From the project directory, execute:

appc run

Once the server starts, you can make cURL or other requests to the server. Open the admin console, then go to the API Docs tab to retrieve the cURL command for the method to find all Tweets. Copy and paste the command in a terminal. The command should print out twenty tweet models.

Publish the Connector

To publish the connector, execute the following command from the project directory:

appc publish

By default, the access level for the connector is set to private, so only the creator can access the connector. To share the connector with other people or publicly, specify a different access level with the appc access command and add people or orgs to your component using the appc user and appc org commands.
