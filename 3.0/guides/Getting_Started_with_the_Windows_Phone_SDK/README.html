Overview

This guide walks through the steps to download and setup the Windows Phone SDK to support Windows Phone app development (ARM-based apps). For Windows Store app development (x86-based apps), see Getting Started with the Windows 8 SDK.

Windows Hybrid

Since Release 3.3.0, you can build Windows Hybrid apps with the Titanium SDK, where your Mobile Web applications are turned into Windows Phone HTML5 projects built with HTML, CSS and JavaScript. Create, build and run projects with the Titanium CLI, then you can debug and package your application in Microsoft Visual Studio.

Windows Platform Beta

The Windows Platform allows you to use the Titanium APIs to create native Windows Phone components, just like with the Android and iOS platforms. Currently, Windows Platform support is only available as a beta on the Titanium SDK and Studio. You need to install 4.0 versions of the Titanium SDK and Studio.

Compatibility and Download

Titanium requires the Window Phone 8 SDK and Microsoft Visual Studio to develop Windows Phone 8 applications.

Windows OS

Microsoft only supports Windows Phone 8 development on systems running 64-bit Windows 8.x. The Pro edition or higher is required to run the Windows Phone emulator.

If you do not have a Windows host system, you can run Windows on either VMWare Fusion or Parallels Desktop for OS X systems, or VMWare Player for Linux systems. Download an evaulation version of Windows 8.1 from http://www.microsoft.com/en-us/evalcenter/evaluate-windows-8-1-enterprise.

Note that if you use VirtualBox, you will not be able to run the Windows Phone emulator since VirtualBox does not support nested virtualization.

Windows Phone 8 SDK

To develop Windows Phone apps, the Windows Phone 8 SDK must be installed.

  • Windows Hybrid requires the Windows Phone 8.0 SDK.

  • Windows Platform requires the Windows Phone 8.1 SDK.

Visual Studio

Windows 8 Phone development requires Visual Studio.

  • Windows Hybrid supports Visual Studio Community or higher, and either the 2012 or 2013 edition.

  • Windows Platform supports Visual Studio 2013, Community edition or higher.

Note that the Express version of Visual Studio is not supported due to a CMake bug.

Windows Phone Emulator

To run the Windows Phone emulator to test your applications, your system must meet the following requirements:

  • 64-bit Windows 8 Pro or higher

  • At least 4 GB of memory

  • Supports hardware virtualization (Intel VT-x or AMD-V)

  • Supports Second Level Address Translation (SLAT)

Windows Dev Center Account

To test and publish applications, you need a Windows Dev Center Account. See https://dev.windowsphone.com/join.

  • Download the Windows Phone SDK from http://dev.windowsphone.com/downloadsdk.

  • Download Visual Studio Community 2013 for free or a trial version of Visual Studio Professional 2013 from http://www.microsoft.com/visualstudio/downloads.

Installation

Before installing any components, make sure your Windows OS is up-to-date with the latest updates. If you have an existing version of Visual Studio installed, make sure it is up-to-date with the latest updates. Check the Updates section from http://www.microsoft.com/visualstudio/downloads.

Setup Visual Studio

  1. Download and install Visual Studio:

    1. Visual Studio needs to be installed to the default location: C:\Program Files (x86)\Microsoft Visual Studio <VERSION>.

    2. If you are using Visual Studio 2013, you have the option of downloading and installing the Windows Phone 8 SDK when you setup Visual Studio.

  2. Install the Windows Phone SDK:

    1. For Visual Studio 2012, download and install the Windows Phone SDK, which installs components to your Visual Studio installation.

    2. For Visual Studio 2013, if you did not select to download the SDK when you installed Studio, you need to modify your installation.

      1. Open Control Panels.

      2. Click Uninstall a Program from the Programs group.

      3. Select Microsoft Visual Studio 2013 and click Change.

      4. When the Visual Studio dialog opens, click Modify.

      5. Select Windows Phone SDK and click Update.

  3. Launch Visual Studio after you have installed the Windows Phone SDK, at least once, before building an application to setup the Visual Studio environment and check for any updates.

Enable Virtualization Technology

Make sure Intel or AMD Virtualization is on before running the Windows Phone emulator for Windows systems.

Many of the steps below may vary depending on your motherboard, processor type, chipset and OEM. Refer to your system's accompanying documentation for the correct information on configuring your system.

  1. Reboot the computer and open the system's BIOS menu. This can usually be done by pressing the delete key, the F1 key or Alt + F4 keys depending on the system.

  2. Enable the virtualization extensions in BIOS:

    1. Open the Processor submenu. The processor settings menu may be in Chipset, Advanced CPU Configuration or Northbridge .

    2. Enable Intel Virtualization Technology (also known as Intel VT-x). The virtualization extensions may be labeled Virtualization Extensions, Vanderpool or various other names depending on the OEM and system BIOS. AMD-V is enabled by default and cannot be disabled in BIOS.

    3. Enable Intel VT-d or AMD IOMMU, if the options are available.

    4. Select Save & Exit .

  3. Reboot the machine.

Running Windows 8 with Parallels

In order to run the Windows Phone emulator on Parallels, you need to configure some additional settings.

  1. Open the Configure dialog.

    1. From the menu, select the Parallels icon or right-click the Parallels Desktop icon and select Control Center.

    2. Click the Configure icon next to the virtual machine.

  2. Click Hardware.

  3. Click CPU & Memory.

  4. In the CPUs drop-down box, change the number of processor cores to 2 or more.

  5. Change Memory to at least 2048 MB.

  6. Make sure Enable nested virtualization is checked.

Running Windows 8 with VMWare

In order to run the Windows Phone emulator on VMWare, you need to configure some additional settings.

  1. Enable hypervisor applications and increase the number of processor cores for the VM:

    1. Open the Settings dialog for the virtual machine. (From the menu, select Virtual Machine > Settings.)

    2. Select Processors & Memory.

    3. In the Processors drop-down, change the number of processor cores to 2 or more.

    4. Expand Advanced options.

    5. Make sure Enable hypervisor applications in this virtual machine is checked.

  2. Edit the config file:

    1. Open the Virtual Machine Library dialog (From the menu, select Window > Virtual Machine Library or Command+Shift+L).

    2. Right-click on your virtual machine.

    3. Hold down the Option key and select Open Config File in Editor to open the configuration file.

    4. Add the following two lines to the end of the file:

      hypervisor.cpuid.v0 = "FALSE"
      mce.enable = "TRUE"

Set PowerShell Permissions

Before executing any commands from the terminal, you need to change the execution policy for PowerShell to run some of the build scripts. This process only needs to be done once.

  1. Open PowerShell as an Administrator:

    1. Go to the Start screen or Apps view.

    2. Start typing "PowerShell".

    3. When the Windows PowerShell icon appears, right-click it and select Run as administrator.

  2. Execute the following command:

    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

The previous command enables the execution of all scripts that are signed by a trusted publisher for the current logged in user.

For more information about this command, see Microsoft TechNet: SetExecution-Policy article.

Using the CLI

Before running any of the CLI commands below, you need to obtain your publisher GUID. After you have created your Windows Dev Center account, log in to https://dev.windowsphone.com/en-us/Account/Summary to get your publish GUID.

Titanium CLI 3.5.x

Starting with Titanium CLI 3.5.x, you need to pass the device ID as part of the CLI -C option (or omit the option and be prompted for the device ID). To get the device ID, run the ti info command. Previously, pass xd for the device emulator or de for a device connected to your host computer.

You can also use the -G option instead of the --wp8-publisher-guid option to pass the publisher GUID to the CLI, or set the windows.phone.publisherGuid configuration option with the appc ti config command.

Run on the Emulator

To build a Windows Phone application on the emulator, run:

## Windows Hybrid
appc run -p mobileweb -T wp8 -C <DEVICE_ID> -G <PUBLISHER_GUID>
 
## Windows Platform
appc run -p windows -C <DEVICE_ID> -G <PUBLISHER_GUID>

After the build process is complete, your application launches automatically on the Windows Phone emulator.

Run on Device

To build a Windows Phone application on device, make sure your device is connected to your host machine and run:

## Windows Hybrid
appc run -p mobileweb -T wp8 -C <DEVICE_ID> -G <PUBLISHER_GUID>
## Windows Platform
appc run -p windows -C <DEVICE_ID> -G <PUBLISHER_GUID> 

While the application is building, you need to keep your phone from locking or the build will fail when trying the install the application to the device.

If this is the first time you are trying to install an application on this device, you will be prompted to unlock it. Follow the on screen directions.

After the build process is complete, the application automatically launches on device.

If you receive an error saying that the Windows Phone IP over USB Transport (IpOverUsbSvc) is not running, check to make sure it is enabled:

  1. Open Administrative Tools.

  2. Double-click Services.

  3. Right-click Windows Phone IP over USB Transport (IpOverUsbSvc) and select either Start or Restart.

Package the Application

For Windows Hybird projects, see the Using Visual Studio section.

For Windows Platform projects, currently, packaging is not implemented.

Using Studio (Preview)

You can use Studio to mange your Windows Platform apps.

Install the Windows Platform Plugin

To use Studio to manage your Windows Phone apps, you need to install the Appcelerator Windows Platform plugin. Before installing the plugin, your current version of Studio must be 3.4.1.

To install the plugin:

  1. From the menu select, Help > Install New Software... to open the Install dialog.

  2. Paste the following URL in the Work with: textbox: http://preview.appcelerator.com/appcelerator/studio/windows/update/beta/

  3. Select the Appcelerator Window Platform Integration item.

  4. Click Next and follow the rest of the instructions.

Configure Windows Support

These steps outline the Studio Preferences settings that need to be changed to support Windows development.

  1. From the menu select, Window > Preferences to open the Preferences dialog.

  2. In Preferences, navigate to Studio > Platform > Windows.

  3. In the Windows section:

    1. Set the path to the Microsoft SDKs folder in the Windows SDK Home textbox. By default, this should automatically be set to C:\Program Files(x86)\Microsoft SDKs. Do not change this value unless you need to.

    2. Set your publisher GUID in the Publisher GUID textbox. This must be set in order to deploy applications to the emulator or device, or package the application for distribution. You can find the GUID in the Summary section of your Windows Dev Center account.

    3. Set your Windows Store certificate in the Windows Store Certificate textbox. This must be set in order to package your application to the Windows Store.

  4. Click OK to save your changes.

Launch the Application

Click on the Run icon and select the Windows Emulator or Device to launch the application on. Titanium builds and installs the application. After installation, the application automatically launches.

Package the Application

Currently, packaging is not implemented for the Windows Platform.

Using Visual Studio

Once you have built you Windows Hybrid project, you can open it in Visual Studio to take advantage of it debugging tools and package it for distribution on the Windows Phone Store.

Open the Project in Visual Studio

  1. From the menu, select File > Open > Project/Solution or press Control + Shift + O to open the Open Project dialog.

  2. Navigate to your project's .\build\mobileweb-wp8 directory.

  3. Select your project's SLN file and click Open.

Update the App Manifest File

In the Solutions Explorer view (equivalent of the App Explorer view in Eclipse-based IDEs), expand the Properties item, then double-click the WMAppManifest.xml file to open it in the Manifest Designer (equivalent of the Editor in Eclipse-based IDEs).

This file is used to manage settings used for deployment, such as specifying icon assets, orientation modes and application permissions.

Package the Application

When you build your application, it generates the XAP file that is uploaded to the Windows Phone Store. To optimize the XAP file, build it in release mode for ARM devices:

  1. From the menu, select Build > Configuration Manager....

  2. In the Configuration Manager dialog, change:

    1. Active solution configuration to Release.

    2. Active solution platform to ARM.

  3. Click Close.

  4. From the menu, select Build > Build Solution or Control + Shift + B.

Once the build is done, the generated XAP file will be in the .\build\mobileweb-wp8\<APP_ID>\Bin\ARM\Release\ directory.

API Usage and Limitations

All of the Mobile Web APIs work for Windows Phone and Store applications with the following exceptions:

  1. You cannot use the HTTP client to make requests back to yourself, that is, you cannot use localhost. Because of this limitation, the RSS Reader sample application does not work.

  2. You cannot use protocol relative URLs, for example, //www.appcelerator.com/support. The default protocol for Windows Phone and Store applications is neither HTTP nor HTTPS.

  3. For Windows Store applications, when using the alert method, if the application currently has an alert dialog open, a new alert cannot be opened until the current one is dismissed.
